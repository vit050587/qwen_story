# AI PD Analyzer — Сервис автоматизированной экспертизы проектной документации

## Краткое описание сервиса

**Цель сервиса**: автоматизированная проверка проектной документации на соответствие требованиям пожарной безопасности, включая анализ ссылок на нормативные документы и проверку чертежей.

**Основные возможности**:
- 📅 Извлечение даты выдачи ГПЗУ (градостроительного плана земельного участка) с помощью компьютерного зрения
- 📚 Автоматическая актуализация нормативной базы на дату выдачи ГПЗУ
- 🔍 Поиск и парсинг ссылок на нормативные документы в проектной документации (МОРВ)
- ✅ Семантическое сравнение контекста использования норм с оригинальным текстом документов
- 📐 **Новое**: Детекция и анализ архитектурных чертежей (размеры, эвакуационные выходы, лестничные клетки и др.)

**Поддерживаемые нормативные документы**:
- Федеральные законы: 123-ФЗ, 69-ФЗ, 384-ФЗ, 184-ФЗ
- Своды правил: СП 1.13130, СП 2.13130, СП 3.13130, СП 4.13130, СП 6-10.13130, СП 54.13330, СП 59.13330, СП 60.13330, СП 113.13330, СП 118.13330, СП 156.13130, СП 253.1325800, СП 256.1325800, СП 477.1325800, СП 484-486.1311500
- ГОСТ: 12.1.004, 31251, 34305, Р 53296, Р 56177
- Постановления Правительства РФ: №87, №1479

## Архитектура и компоненты

### Backend
- **Flask** REST API с поддержкой сессий обработки
- **Асинхронная обработка** в фоновых потоках с отслеживанием статуса
- **Персистентность** сессий в JSON-файле
- **Docker Compose**: 2 сервиса (web + ollama)

### Frontend
- Веб-интерфейс для загрузки файлов (ГПЗУ + МОРВ)
- Отслеживание статуса обработки в реальном времени
- Предпросмотр результатов (JSON/XLSX/CSV)
- Скачивание отчетов в формате Excel

### Конвейер обработки (6 этапов)

| Этап | Описание | Модель/Инструмент |
|------|----------|-------------------|
| 1 | Извлечение даты из ГПЗУ | Qwen2.5-VL (OCR) |
| 2 | Актуализация норм на дату ГПЗУ | YandexGPT-5 + pandas (XLSX) |
| 3 | Копирование актуальных норм | Файловая система |
| 4 | Поиск ссылок в МОРВ | YandexGPT-5 |
| 5 | Парсинг номеров пунктов | YandexGPT-5 |
| 6 | Сравнение контекста | GPT-OSS:20b |
| **7 (новый)** | **Детекция чертежей** | **PyMuPDF (размер >30 см)** |
| **8 (новый)** | **Анализ чертежей** | **Qwen3.6 (VLM)** |
| **9 (новый)** | **Валидация параметров** | **Gemma3:27b** |

## Используемые LLM-модели

| Модель | Назначение | Конфигурация |
|--------|-----------|--------------|
| `qwen2.5vl:7b` | OCR ГПЗУ (поиск заголовка и даты) | temperature=0.1, timeout=120s |
| `yandex/YandexGPT-5-Lite-8B-instruct-GGUF:latest` | Извлечение ссылок, актуализация норм, парсинг пунктов | temperature=0.1, timeout=1200s |
| `gpt-oss:20b` | Семантическое сравнение контекста | temperature=0.1, timeout=300s |
| `qwen3.6:latest` **(новая)** | Анализ изображений чертежей (13 параметров) | temperature=0.0, num_predict=2000 |
| `gemma3:27b` **(новая)** | Валидация параметров чертежей на соответствие СП 1.13130 | temperature=0.0, num_predict=1500 |

**Переменные окружения для настройки моделей**:
```bash
NORMS_LLM_MODEL=yandex/YandexGPT-5-Lite-8B-instruct-GGUF:latest
DRAWING_VLM_MODEL=qwen3.6:latest
DRAWING_VALIDATION_MODEL=gemma3:27b
DRAWING_MIN_SIZE_CM=30.0
```

## API Endpoints

### Сессии
| Метод | Endpoint | Описание |
|-------|----------|----------|
| GET | `/fire/api/sessions` | Список всех сессий |
| GET | `/fire/api/session/<id>` | Полные данные сессии |
| GET | `/fire/api/session/<id>/status` | Краткий статус |
| DELETE | `/fire/api/session/<id>` | Удаление сессии |
| POST | `/fire/api/session/<id>/restore` | Восстановление сессии |

### Загрузка файлов
| Метод | Endpoint | Описание |
|-------|----------|----------|
| POST | `/fire/api/upload_first` | Загрузка ГПЗУ (возвращает sessionId) |
| POST | `/fire/api/upload_second?sessionId=<id>` | Загрузка МОРВ |

### Файлы
| Метод | Endpoint | Описание |
|-------|----------|----------|
| GET | `/fire/api/session/<id>/download/<filename>` | Скачать файл |
| GET | `/fire/api/session/<id>/preview/<filename>` | Предпросмотр (JSON/XLSX/CSV) |

### Статусы сессии
- `processing_norms` — обработка ГПЗУ и актуализация норм
- `awaiting_second` — готов к загрузке МОРВ
- `processing_second` — обработка МОРВ (поиск ссылок, сравнение, анализ чертежей)
- `completed` — завершено, результаты готовы
- `error` — ошибка обработки

## Результаты обработки

### Выходные файлы
1. **все_СП_результаты.xlsx** — актуальные версии нормативных документов на дату ГПЗУ
2. **ФЗ_*.xlsx** — проверка ссылок на федеральные законы (123-ФЗ, 69-ФЗ и др.)
3. **СП_*.xlsx** — проверка ссылок на своды правил (СП 1.13130 и др.)
4. **ГОСТ_*.xlsx** — проверка ссылок на ГОСТ
5. **ПП_*.xlsx** — проверка ссылок на постановления Правительства
6. **drawings_report_<sessionId>.xlsx** *(новый)* — отчет по анализу чертежей

### Структура отчета по чертежам
| Параметр | Описание | Статус |
|----------|----------|--------|
| Наличие балконов/лоджий | Проверка обязательности для секционных домов | ok/not_ok/not_applicable |
| Количество лестничных клеток | Минимум 2 для этажа >500м² или >12 квартир | ok/not_ok/not_applicable |
| Расстояние до выхода | Не более нормируемого значения | ok/not_ok/not_applicable |
| Ширина коридора | Не менее 1.4м для жилых помещений | ok/not_ok/not_applicable |
| Ширина лестничного марша | Не менее 1.05м | ok/not_ok/not_applicable |
| Высота здания | Классификация по высоте | ok/not_ok/not_applicable |
| Тип этажа | Корректность определения (жилой/технический/парковка) | ok/not_ok/not_applicable |

## Быстрый старт

### Требования
- Docker и Docker Compose
- NVIDIA GPU (для работы LLM-моделей)
- NVIDIA Container Toolkit

### Установка и запуск

```bash
# Клонирование репозитория
git clone <repository-url>
cd ai-pd-analyzer

# Настройка переменных окружения
cp .env.example .env
# Отредактируйте .env при необходимости

# Запуск сервисов
docker-compose up -d

# Проверка логов
docker-compose logs -f web
docker-compose logs -f ollama
```

Сервис доступен по адресу: `http://localhost:6000`

### Остановка
```bash
docker-compose down
```

## Производительность и ограничения

| Параметр | Значение |
|----------|----------|
| Размер входных файлов | Не ограничен |
| Среднее время обработки | ~40 минут (зависит от объема МОРВ) |
| Таймаут OCR ГПЗУ | 120 секунд на страницу |
| Таймаут извлечения ссылок | 1200 секунд (20 минут) |
| Таймаут сравнения контекста | 300 секунд (5 минут) |
| Таймаут анализа чертежа | 120 секунд на страницу |

**Рекомендации**:
- Для больших проектов (>500 страниц) время обработки может достигать 1-2 часов
- Чертежи обрабатываются последовательно с паузой 2 секунды между запросами
- При ошибках таймаута используется fallback на YandexGPT-5

## Примеры запросов

### 1. Загрузка ГПЗУ
```bash
curl -X POST http://localhost:6000/fire/api/upload_first \
  -F "file=@gpzu.pdf"
```

**Ответ**:
```json
{
  "sessionId": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
  "extractedDate": "15.01.2023",
  "status": "processing_norms",
  "message": "Файл принят. Начинаем актуализацию нормативной базы…"
}
```

### 2. Загрузка МОРВ

```bash
curl -X POST "http://localhost:6000/fire/api/upload_second?sessionId=3fa85f64-5717-4562-b3fc-2c963f66afa6" \
  -F "file=@mopb.pdf"
```

### 3. Проверка статуса
```bash
curl http://localhost:6000/fire/api/session/3fa85f64-5717-4562-b3fc-2c963f66afa6/status
```
