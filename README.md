# ai-pd-analyzer_5

## Краткое описание сервиса

**Цель сервиса**: выявление несоответствия использованных в МРОВ ссылок на нормативную документацию (Федеральный закон №123-ФЗ, СП 1.13130, СП 2.13130, СП 3.13130, СП 4.13130 и др.) согласно контексту использования.  

**Задачи**: 
- Определение даты выдачи ГПЗУ;
- Определение актуального набора нормативных документов, действующих на момент выдачи ГПЗУ;
- Выявление ссылок на соответствующие нормативные документы в тексте проектной документации;
- Сравнение контекста применения статьи нормативного документа в тексте проекта.


В файле `main.py` собрана функция `process_mopb_validation`, которая обрабатывает два PDF файла (ГПЗУ или файл, содержащий ГПЗУ, и файл проектной документации) и возвращает JSON с результатами.

Аргументы:  
- `gpzu_pdf_path`: путь к первому PDF файлу (ГПЗУ)   
- `mopb_pdf_path`: путь ко второму PDF файлу (МОРВ)  
- `output_dir`: директория для сохранения результатов   (установлено значение по умолчанию)

Возвращает:  
Словарь с ключами:  
- `extracted_date`: извлеченная дата из ГПЗУ  
- `files`: список словарей с информацией о сгенерированных файлах  

Каждый файл:  
```json
{
  "path": "...",
  "filename": "...",
  "size": "..."
}
```

Необходимо, чтобы сначала обрабатывался файл ГПЗУ, затем выводилась дата, а уже затем можно было загрузить файл проектной документации и начиналась его обработка.
На выходе должно быть возможно скачать файлы обработанного документа (в формате csv и txt)

## Используемые модели

В сервисе используется три модели: 
1. `Qwen2.5-vl:7b` - для обработки страниц ГПЗУ, если текст на старницах не распознается pdf ридерами
2. `YandexGPT-5-Lite:8b` - для обнаружения ссылок в тексте проектной документации, поиск нормативных документов актуальных на дату выдачи ГПЗУ, а также используется как fallback при timeout ошибках при обращении к `GPT-OSS:20b`
3. `GPT-OSS:20b` - для сравнения текста проектной документации с соответствующей статьей/пунктом нормативного документа

## Производительность и ограничения
Размер входных данных не ограничен

Примерное время обработки запроса составляет около 40 минут

Таймауты при обращении к моделям составляют:
- 5 минут при сравнении с использованием GPT-OSS
- 2 минуты при извлечении ссылок из текста проектной документации
